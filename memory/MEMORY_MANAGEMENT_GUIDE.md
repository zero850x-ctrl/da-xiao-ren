# 記憶管理指南

## 核心原則

**不要載入整個文件，只搜索需要的內容！**

## 工具使用流程

### 1. 記憶搜索 (`memory_search`)
當你需要查找特定信息時：
```javascript
memory_search({
  query: "要找的關鍵詞或問題",
  maxResults: 5
})
```

### 2. 獲取具體內容 (`memory_get`)
搜索後，只載入需要的部分：
```javascript
memory_get({
  path: "找到的文件路徑",
  from: 開始行數,
  lines: 需要的行數
})
```

## 使用場景示例

### 場景1：查找過去的決策
```
問題：我們之前關於文件管理的決定是什麼？

步驟：
1. memory_search({query: "文件管理 決定"})
2. 從結果中找到相關片段
3. memory_get({path: "memory/2026-02-08.md", from: 10, lines: 5})
```

### 場景2：查找人物信息
```
問題：Gordon 的偏好是什麼？

步驟：
1. memory_search({query: "Gordon 偏好"})
2. 從 USER.md 或 MEMORY.md 中獲取相關行
```

### 場景3：查找項目歷史
```
問題：我們之前做過什麼項目？

步驟：
1. memory_search({query: "項目 完成"})
2. 從多個文件中收集相關信息
```

## 最佳實踐

1. **先搜索，後載入** - 永遠先使用 `memory_search`
2. **只載入需要的部分** - 使用 `memory_get` 的精確控制
3. **保持查詢具體** - 使用具體的關鍵詞
4. **限制結果數量** - 通常 3-5 個結果就足夠了

## 避免的錯誤

❌ **錯誤做法**：直接讀取整個大型文件
❌ **錯誤做法**：載入所有歷史文件
✅ **正確做法**：搜索特定信息，只載入相關片段

## 定期維護

1. **每月清理** - 移動大型文件到 archive/
2. **更新索引** - 確保重要信息在 MEMORY.md 中
3. **優化結構** - 保持文件組織清晰
