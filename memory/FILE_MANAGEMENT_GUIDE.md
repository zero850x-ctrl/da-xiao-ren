# 文件管理指南

## 目錄結構

```
memory/
├── daily/          # 每日簡短記錄 (< 100 行)
├── weekly/         # 每週總結
├── monthly/        # 每月報告
├── archive/        # 存檔的大型文件
├── logs/           # 系統日誌和詳細記錄
├── projects/       # 項目相關文件
└── FILE_MANAGEMENT_GUIDE.md
```

## 文件大小限制

- **每日文件**: < 100 行
- **每週總結**: < 200 行  
- **每月報告**: < 500 行
- **任何超過 200 行的文件** 應該移到 `archive/` 目錄

## 自動載入規則（AGENTS.md）

根據 AGENTS.md 的更新，系統只會自動載入：
1. `SOUL.md` - 你的身份
2. `USER.md` - 用戶信息
3. **小型的記憶文件** (< 200 行)
4. `MEMORY.md` - 長期記憶（僅在主會話）

## 最佳實踐

1. **保持文件簡潔** - 只記錄重要信息
2. **使用子目錄** - 按類別組織文件
3. **定期清理** - 每月運行清理腳本
4. **使用搜索工具** - 用 `memory_search` 查找特定信息

## 清理腳本

使用 `cleanup_large_files.sh` 定期清理大型文件：
```bash
./cleanup_large_files.sh
```
